💳 Real-Time Insurance Eligibility

DischargeIQ integrates with the Stedi clearinghouse to perform live X12 270/271 eligibility checks. Configure the following environment variables to enable real-time verification.

ELIGIBILITY_ENABLED
Master toggle. Set to true to enable eligibility checks on plan generation.
—
ELIGIBILITY_MOCK
Set to true to use deterministic mock data instead of calling Stedi. Safe for testing.
—
STEDI_API_KEY
Stedi clearinghouse API key. Obtain from app.stedi.com → API Keys.
—
HOSPITAL_NPI
Your facility's 10-digit NPI. Used as the rendering provider in eligibility requests.
—
# .env — add these to enable live eligibility
ELIGIBILITY_ENABLED=true
ELIGIBILITY_MOCK=false
STEDI_API_KEY=your-stedi-api-key-here
HOSPITAL_NPI=1234567890

🔒 Security & HIPAA

• The eligibility_cache table stores structured eligibility results and is PHI — ensure your PostgreSQL provider has a signed HIPAA BAA.

• Raw eligibility responses (X12 271) are never logged or stored — only structured fields are persisted.

• Audit logs record only: timestamp, payer_id, check duration (ms), is_eligible (boolean), and source. No member IDs, names, or DOB are logged.

• STEDI_API_KEY is read from environment only — never stored in the database or committed to source control.

• Cache TTL is 4 hours. Eligibility status can change; always verify before acting on cached results.
